🌐 1. IP Address (Identity of a Machine)

An IP address is like a home address for a device on a network.

Example
192.168.1.10 (private)
8.8.8.8 (public)
Types
Public IP → accessible from internet
Private IP → used inside network only
Diagram (think like this)
Internet
   ↓
[Public IP: 52.10.x.x]
   ↓
[Private IP: 10.0.0.4] → Your VM

👉 Every VM in Microsoft Azure / Amazon Web Services gets an IP.

🌍 2. DNS (Domain Name System)

DNS converts names → IP addresses.

Example
You type: google.com
DNS gives: 142.250.x.x
Why important?

Humans remember names, not numbers.

Diagram
You → google.com
        ↓
     DNS Server
        ↓
   142.250.x.x
        ↓
     Website loads

👉 Without DNS, you'd need to remember IPs for every site 😅

🏠 3. VPC / VNet (Your Private Network in Cloud)
In AWS → AWS VPC
In Azure → Azure Virtual Network

This is your own isolated network in the cloud.

What it does
You control IP ranges
You isolate your resources
You decide who can talk to what
Diagram
Cloud
 └── Your VPC (10.0.0.0/16)
      ├── VM1
      ├── VM2
      └── Database

👉 Think of it as your private colony inside the cloud

🧱 4. Subnet (Smaller Network Inside VPC)

A subnet divides your network into smaller parts.

Why?
Better organization
Better security
Example
VPC: 10.0.0.0/16

  ├── Public Subnet (10.0.1.0/24)
  │     └── Web Server
  │
  └── Private Subnet (10.0.2.0/24)
        └── Database

👉 Public = internet access
👉 Private = internal only

🚦 5. Routing (Traffic Direction)

Routing decides where traffic goes.

Example
Internet traffic → goes through gateway
Internal traffic → stays inside network
Diagram
VM → Route Table → Internet Gateway → Internet

👉 Like Google Maps deciding your route

🔐 6. Security Groups / Firewalls

These control who can access what.

Example Rules
Allow port 80 (HTTP)
Allow port 22 (SSH)
Block everything else
Diagram
User → [Firewall Rules] → VM
        ✔ allow 80
        ❌ block others

👉 Most cloud hacks = bad firewall rules

🌉 7. Internet Gateway

This allows your private network to talk to the internet.

Diagram
VPC → Internet Gateway → Internet

👉 Without it, your VM is completely private

⚖️ 8. Load Balancer

Distributes traffic across multiple servers.

Example
1000 users → split across 3 servers
Diagram
Users
  ↓
Load Balancer
 ├── VM1
 ├── VM2
 └── VM3

👉 Prevents crashes & improves performance

🔁 9. NAT Gateway (Private → Internet Access)

Allows private servers to access internet (but not be accessed).

Example
Database downloading updates
Diagram
Private VM → NAT Gateway → Internet
(No inbound access allowed)

Concept	Simple Meaning
